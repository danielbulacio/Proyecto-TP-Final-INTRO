¿Cómo doy de alta una nueva parcela y qué datos requiere?
expand_more

Para registrar una parcela debes acceder a la pestaña Parcelas e ingresar un nombre descriptivo junto con su latitud y longitud en formato decimal.

Estos datos se guardan en la tabla parcelas de tu base de datos relacional:

Tabla: parcelas (nombre, latitud, longitud)
cloud_queue
¿Cómo se actualizan los datos de clima mediante OpenMeteo?
expand_more

La aplicación utiliza las coordenadas geográficas de la parcela (latitud y longitud) para hacer una petición externa a la API gratuita de OpenMeteo.

Esta llamada recupera automáticamente la temperatura actual, el nivel de precipitaciones y la humedad del suelo de la zona, almacenándolos en el historial.

Tabla asociada: detalle_parcela
water_drop
¿Qué es el índice de Evapotranspiración registrado en los detalles?
expand_more

La evapotranspiración representa la pérdida de humedad de una parcela debido a la evaporación directa del suelo sumada a la transpiración de las plantas. Es un dato crítico que provee la API de OpenMeteo para calcular si necesitas activar sistemas de riego adicionales.

Campo: detalle_parcela.evapotranspiracion
grass
¿Cómo asocio un nuevo cultivo a una parcela que ya creé?
expand_more

Desde la sección Cultivos, puedes añadir una plantación especificando su parcela mediante un selector. Internamente, la base de datos utiliza una clave foránea que relaciona el id del cultivo con el id de la parcela de destino.

Relación: cultivos.parcela_id -> parcelas.id
thermostat
¿Para qué sirve configurar la "Temperatura Óptima" o "Mililitros"?
expand_more

Al registrar a mano parámetros como la temperatura óptima o los mililitros de agua diarios requeridos, el sistema puede alertarte si los datos de clima de OpenMeteo están por debajo o por encima de lo que tu siembra necesita para crecer sana.

Campos: temperatura_optima, mililitros_necesarios
assignment_turned_in
¿Cómo llevo el control de tareas y su estado actual?
expand_more

En el módulo Tareas, puedes crear labores específicas como riego, abono o poda para cada parcela. El campo hecho se guarda como un booleano (falso/verdadero). Al marcar la casilla "Completado" en la interfaz, el estado se actualiza dinámicamente en tu base de datos.

Tabla: tareas (tarea, hecho)
playlist_play
¿Se pueden planificar movimientos a futuro en la agenda?
expand_more

¡Sí! Las tareas se registran con el estado FALSE (pendiente) de manera inicial. Esto te permite programar tareas futuras, organizar el orden de actividades en el campo y luego tildar las completadas a medida que se efectúen.
